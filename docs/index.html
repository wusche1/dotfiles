tmux + Claude Code

A take-home guide for running terminal coding sessions

tmux keeps your work alive in the background. Close the terminal, sleep the laptop, or drop an SSH connection — your session (and Claude) keeps running. You just reattach. Almost every command is the prefix Ctrl+Space (press and release), then a key.

The mental model

tmux nests three layers. Give each one a job and you always know where things live.

Session = project. One per repo. The session is the container that stays alive when you detach and walk away.

Window = branch. Inside a project, one window per branch / git worktree. Claude can refactor on one branch while another sits untouched in the next window.

Pane = agent vs. your eyes. Split a window so the agent works in one pane while you watch the code, run tests, or read a diff in the other.

In one line: session = which project, window = which branch, pane = agent vs. your view of the code.

Survival keys

If you remember nothing else, remember these.

Ctrl+SpacethendDetach — leave everything running and walk away
tmux attachReattach to your running sessions
tmux lsList what is currently running
Ctrl+Spacethen[Scroll up to read history — arrows to move, q to quit

Windows = branches

Like tabs. One per branch / worktree. The bottom bar is your branch list.

Ctrl+SpacethenWCreate a git worktree + window for a new branch in one step
Ctrl+SpacethencNew window (same branch)
Ctrl+Spacethenn / pNext / previous window
Ctrl+Spacethen1 2 3Jump to window by number
Ctrl+Spacethen,Rename the current window
exitClose a window (just exit its last shell)

Panes = code + agents

Split the screen so the agent and your view of the code sit side by side.

Ctrl+SpacethenvSplit left/right (e.g. Claude left, terminal right)
Ctrl+SpacethensSplit top/bottom
Ctrl+Spacethenh j k lMove between panes (vim directions)
exitClose a pane

Sessions = projects

One per repo. Switching projects means switching sessions, not closing them.

Ctrl+SpacethenSNew named session in ~/Projects
Ctrl+Spacethen/Quick-switch between sessions, then press a number

Copy & paste

Mouse selection often works too — this is the keyboard way.

Ctrl+Spacethen[Enter scroll/copy mode
vStart selecting (then move with vim keys)
yCopy selection and exit
Ctrl+Spacethen]Paste

Gotchas

F12 toggles tmux's keys off, for when you SSH into another tmux. If the prefix seems to stop working and you see [OFF] in the bottom-right, press F12 to turn it back on.

Mouse is on: click panes/windows to switch, drag borders to resize, scroll to read back.

A typical Claude Code session

1. tmux attach — or start fresh by running tmux

2. Ctrl+Space S → name the session after your project

3. Run claude

4. Need a scratch terminal? Ctrl+Space v

5. New task / branch? Ctrl+Space W spins up a worktree + window

6. Walking away? Ctrl+Space d — Claude keeps working

7. Back later? tmux attach
